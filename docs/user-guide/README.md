# DataCentral Pulse — User Guide (source)

The screenshot-driven, two-persona user guide for Pulse. Published as a Claude
Artifact, but fully self-contained — the built file opens in any browser with no
network access.

## Files

| File | What it is |
|------|------------|
| `guide.html` | **The source you edit.** The `<style>` design system + all guide content, with `{{IMG:name}}` placeholders where screenshots go. Uses normal UTF‑8 punctuation — readable and diff-friendly. |
| `build.mjs` | Build script: inlines each `shots/*.jpeg` as a base64 data URI and entity-encodes the output to pure ASCII. |
| `pulse-user-guide.html` | **The built, publishable output** (~1.9 MB, screenshots embedded). Regenerated by `build.mjs`; don't hand-edit. |
| `shots/` | The 12 screenshots (captured live from the app in both the Customer and DataCentral personas). |
| `research/` | The code-cited research the guide was written and fact-checked against — every claim maps to a `file:line` in the repo. Includes `gap-report.md` (contradictions found and resolved). |

## Rebuild

```bash
cd docs/user-guide
node build.mjs
```

Output: `pulse-user-guide.html`.

## Republish (Claude Code)

Ask Claude to publish `docs/user-guide/pulse-user-guide.html` as an artifact. To
update the **existing** artifact instead of minting a new URL, give Claude the
current artifact URL.

## Editing notes

- **Content and design** live in `guide.html`. Screenshots are referenced by
  token, e.g. `{{IMG:customer-04-submit}}` → `shots/customer-04-submit.jpeg`.
- **Two-persona colour system:** teal (`--cust`) = Customer, violet (`--dc`) =
  DataCentral. Applied as information (nav grouping, matrix columns, section
  rails), not decoration.
- **New screenshot:** drop `shots/<name>.jpeg`, reference it as `{{IMG:<name>}}`,
  rebuild.
- **Accuracy:** the guide is written against `research/`. If the app changes,
  update the relevant research note and the guide together.
- The build output is intentionally pure-ASCII (it has no `<head>`/charset when
  wrapped by the Artifact host); keep new punctuation as normal UTF‑8 in
  `guide.html` — `build.mjs` encodes it.
